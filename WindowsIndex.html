Grafana → PDF on Windows

A complete walkthrough: install Grafana OSS on Windows, build a dashboard with CSV-backed tabular and time-series data, then wire up the free grafana-dashboard-reporter-app plugin to export pixel-perfect PDFs.

Grafana ≥ 10.4 100% Free / OSS Windows 10 / 11 / Server

Architecture Overview

Understand the data flow first — everything else follows from this.

CSV / TestData
→
Grafana OSS
→
grafana-image-renderer (svc)
→
dashboard-reporter-app (plugin)
→
📄 PDF
grafana-image-renderer

Standalone Windows binary that runs headless Chromium. Grafana calls it to render panels as PNG images. Required dependency for the PDF reporter.

dashboard-reporter-app

Unsigned Grafana plugin (mahendrapaipuri). Stitches panel PNGs into an HTML template, then prints to PDF via Chromium. No LaTeX, no separate web server.

Phase 1

Install Grafana & Build the Dashboard

1 Install Grafana on Windows

Download & Install

PowerShell — run as Administratorpwsh
# 1. Download latest Grafana OSS installer (adjust version as needed)
$ver = "12.0.1"
$url = "https://dl.grafana.com/oss/release/grafana-$ver.windows-amd64.msi"
Invoke-WebRequest $url -OutFile "grafana-$ver.msi"

# 2. Silent install
Start-Process msiexec.exe -Wait -ArgumentList '/i',"grafana-$ver.msi",'/quiet'

# 3. Start the Grafana Windows service
Start-Service -Name "Grafana"

# Default install path: C:\Program Files\GrafanaLabs\grafana\

Open http://localhost:3000 — default credentials are admin / admin.

2 Add a Datasource (TestData DB)

We use Grafana's built-in TestData datasource so no external database is needed — ideal for demos and local dev.

Via the UI

  1. Go to Connections → Data sources → Add data source
  2. Search for and select "TestData"
  3. Click Save & test — it should show a green checkmark
ℹ️
TestData ships with every Grafana install and supports scenarios: random walk, CSV metric values, table data, logs, annotations, and more. No network, no auth.

3 Build the Sample Dashboard

We'll provision a dashboard JSON directly — this is faster and reproducible.

3a. Create the provisioning directory

PowerShellpwsh

New-Item -ItemType Directory -Force -Path "$base\dashboards"
New-Item -ItemType Directory -Force -Path "$base\datasources"

Save as …\provisioning\datasources\testdata.yml

Save as …\provisioning\dashboards\local.yml

Phase 2

Install grafana-image-renderer (Windows Binary)

4 Download & Run the Renderer Service

⚠️
Do not use grafana-cli plugins install grafana-image-renderer — the plugin mode is deprecated as of September 2025. Use the standalone binary service.

4a. Download the Windows binary

PowerShell (Admin)pwsh
# Download latest release from GitHub (check https://github.com/grafana/grafana-image-renderer/releases)
$rendererVer = "5.0.2"  # update to latest
$url = "https://github.com/grafana/grafana-image-renderer/releases/download/v$rendererVer/grafana-image-renderer-$rendererVer-windows-x64.zip"
Invoke-WebRequest $url -OutFile "renderer.zip"

Expand-Archive "renderer.zip" -DestinationPath "C:\grafana-renderer"

4b. Create a config file

Save as C:\grafana-renderer\config.json

config.jsonJSON
{
  "service": {
    "host": "localhost",
    "port": 8081
  },
  "rendering": {
    "args": ["--no-sandbox", "--disable-setuid-sandbox"],
    "timeout": 30,
    "mode": "default"
  },
  "security": {
    "authToken": "my-renderer-secret-token"
  }
}

4c. Run as a Windows Service (NSSM)

PowerShell (Admin) — install NSSM first from nssm.ccpwsh
# Using NSSM (Non-Sucking Service Manager) to wrap the renderer EXE
nssm install GrafanaRenderer "C:\grafana-renderer\grafana-image-renderer.exe"
nssm set GrafanaRenderer AppParameters "server --config=C:\grafana-renderer\config.json"
nssm set GrafanaRenderer AppDirectory "C:\grafana-renderer"
nssm set GrafanaRenderer Start SERVICE_AUTO_START
nssm set GrafanaRenderer ObjectName LocalSystem
Start-Service GrafanaRenderer
✅
Verify the renderer is up: Invoke-WebRequest http://localhost:8081 — you should get a 200 response or renderer info page.

5 Configure grafana.ini to Use the Renderer

Edit C:\Program Files\GrafanaLabs\grafana\conf\grafana.ini

grafana.ini — relevant sectionsINI
[rendering]
server_url = http://localhost:8081/render
callback_url = http://localhost:3000/
renderer_token = my-renderer-secret-token
concurrent_render_request_limit = 10

[feature_toggles]
# Required for reporter plugin on Grafana > 10.4.3
enable = accessControlOnCall externalServiceAccounts

[plugins]
# Allow the unsigned reporter plugin to load
allow_loading_unsigned_plugins = mahendrapaipuri-dashboardreporter-app
ℹ️
The renderer_token in grafana.ini must match the authToken value in the renderer's config.json.
Phase 3

Install & Use the Dashboard Reporter (PDF) Plugin

6 Install the Reporter Plugin

6a. Download the plugin zip

PowerShell (Admin)pwsh
# Grab latest release from GitHub
$pluginVer = "1.8.0"  # check github.com/mahendrapaipuri/grafana-dashboard-reporter-app/releases
$url = "https://github.com/mahendrapaipuri/grafana-dashboard-reporter-app/releases/download/v$pluginVer/mahendrapaipuri-dashboardreporter-app-$pluginVer.zip"
Invoke-WebRequest $url -OutFile "reporter-plugin.zip"

# Extract into Grafana's plugins directory
$pluginsDir = "C:\Program Files\GrafanaLabs\grafana\data\plugins"
Expand-Archive "reporter-plugin.zip" -DestinationPath $pluginsDir

# Confirm folder name matches expected plugin ID
Rename-Item "$pluginsDir\mahendrapaipuri-dashboardreporter-app-$pluginVer" `
            "$pluginsDir\mahendrapaipuri-dashboardreporter-app" `
            -ErrorAction SilentlyContinue

7 Enable & Configure the Plugin

7a. Restart Grafana

PowerShell (Admin)pwsh
Restart-Service -Name "Grafana"

7b. Enable via Grafana UI

  1. Log in as admin at http://localhost:3000
  2. Go to Apps → Dashboard Reporter App → Configuration
  3. Toggle the plugin to Enabled
  4. Set Grafana App URL → http://localhost:3000
  5. Under Layout, choose Grid (mirrors dashboard layout) or Simple
  6. Click Save
⚠️
You will see "Invalid plugin signature" in Grafana logs — this is expected for community plugins. It is non-fatal and controlled by the allow_loading_unsigned_plugins setting you added in Step 5.

7c. (Optional) Service Account Token for Grafana > 10.4.4

If the externalServiceAccounts feature flag cannot be enabled, create a manual service account token:

  1. Administration → Service accounts → Add service account
  2. Role: Viewer (read-only is sufficient)
  3. Generate a token and paste it into the reporter plugin configuration page under Service Account Token

8 Generate Your First PDF

Method A — Browser URL (simplest)

Open this URL in your browser while logged in as admin:

Browser Address Bar
http://localhost:3000/api/plugins/mahendrapaipuri-dashboardreporter-app/resources/report?dashUid=sales-demo-001

The browser will download a PDF of the full dashboard.

Method B — PowerShell / curl (for automation)

PowerShellpwsh
$grafanaUrl = "http://localhost:3000"
$dashUid   = "sales-demo-001"
$token     = "Bearer <your-service-account-token>"

$reportUrl = "$grafanaUrl/api/plugins/mahendrapaipuri-dashboardreporter-app/resources/report" `
           + "?dashUid=$dashUid" `
           + "&from=now-7d&to=now" `
           + "&includePanelDataID=3"  # panel 3 = the table panel

Invoke-WebRequest -Uri $reportUrl `
  -Headers @{ "Authorization" = $token } `
  -OutFile ".\SalesReport-$(Get-Date -Format 'yyyyMMdd').pdf"

Write-Host "PDF saved." -ForegroundColor Green
✅
The includePanelDataID=3 parameter tells the reporter to also append the raw tabular data from panel 3 (the Sales Summary table) at the end of the PDF, in addition to its visual screenshot.
