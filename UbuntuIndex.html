Grafana → PDF on Ubuntu 24.04

Same stack as the Windows guide — Grafana OSS, TestData datasource, grafana-image-renderer, and the mahendrapaipuri dashboard reporter — now running natively on Ubuntu 24.04 LTS using apt, systemd, and proper Linux service patterns.

Grafana ≥ 10.4 100% Free / OSS Ubuntu 24.04 LTS systemd services

Phase 1

Install Grafana & Build the Dashboard

1 Install Grafana via APT

1a. Add the Grafana APT repository

bash — run with sudobash
# Install prerequisites
sudo apt-get update
sudo apt-get install -y apt-transport-https software-properties-common wget curl gnupg
    
# Add Grafana GPG key
sudo mkdir -p /etc/apt/keyrings
wget -q -O - https://apt.grafana.com/gpg.key \
  | gpg --dearmor \
  | sudo tee /etc/apt/keyrings/grafana.gpg > /dev/null

# Add the stable OSS repository
echo "deb [signed-by=/etc/apt/keyrings/grafana.gpg] https://apt.grafana.com stable main" \
  | sudo tee /etc/apt/sources.list.d/grafana.list

# Install Grafana OSS
sudo apt-get update
sudo apt-get install -y grafana

1.a.a Fix grafana package not found.

1b. Enable & start the service

Phase 2

Install grafana-image-renderer (Linux Binary)

4 Download & Install the Renderer

⚠️
Use the standalone binary service from GitHub Releases — not the deprecated grafana-cli plugins install grafana-image-renderer plugin form.

4a. Install Chromium dependencies

The renderer ships its own Chromium, but it needs system libraries on Ubuntu.

bashbash
sudo apt-get install -y \
  libx11-6 libx11-xcb1 libxcb1 libxcomposite1 libxcursor1 \
  libxdamage1 libxext6 libxfixes3 libxi6 libxrandr2 libxrender1 \
  libxss1 libxtst6 libnss3 libatk1.0-0 libatk-bridge2.0-0 \
  libcups2 libdrm2 libgbm1 libasound2t64 libpangocairo-1.0-0 \
  libpango-1.0-0 libcairo2 libgdk-pixbuf2.0-0 libgtk-3-0 \
  fonts-liberation libappindicator3-1 xdg-utils

4b. Download & extract the Linux binary

bashbash
# Check https://github.com/grafana/grafana-image-renderer/releases for latest version
RENDERER_VER="5.0.2"
wget -q "https://github.com/grafana/grafana-image-renderer/releases/download/v${RENDERER_VER}/grafana-image-renderer-${RENDERER_VER}-linux-x64.tar.gz" \
  -O /tmp/renderer.tar.gz

sudo mkdir -p /opt/grafana-renderer
sudo tar -xzf /tmp/renderer.tar.gz -C /opt/grafana-renderer --strip-components=1

4c. Create a dedicated service user & fix permissions

bashbash
sudo useradd --system --no-create-home --shell /usr/sbin/nologin grafana-renderer
sudo chown -R grafana-renderer:grafana-renderer /opt/grafana-renderer

4d. Write the renderer config

bashbash
sudo tee /opt/grafana-renderer/config.json <<'EOF'
{
  "service": {
    "host": "localhost",
    "port": 8081
  },
  "rendering": {
    "args": ["--no-sandbox", "--disable-setuid-sandbox"],
    "timeout": 30,
    "mode": "default"
  },
  "security": {
    "authToken": "my-renderer-secret-token"
  }
}
EOF
sudo chown grafana-renderer:grafana-renderer /opt/grafana-renderer/config.json

4e. Create a systemd unit (replaces NSSM)

bashbash
sudo tee /etc/systemd/system/grafana-renderer.service <<'EOF'
[Unit]
Description=Grafana Image Renderer Service
After=network.target
Wants=network.target

[Service]
Type=simple
User=grafana-renderer
Group=grafana-renderer
WorkingDirectory=/opt/grafana-renderer
ExecStart=/opt/grafana-renderer/grafana-image-renderer server \
  --config=/opt/grafana-renderer/config.json
Restart=on-failure
RestartSec=5s
StandardOutput=journal
StandardError=journal
SyslogIdentifier=grafana-renderer

# Hardening
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
EOF
bashbash
sudo systemctl daemon-reload
sudo systemctl enable grafana-renderer
sudo systemctl start grafana-renderer

# Verify
sudo systemctl status grafana-renderer
curl -s http://localhost:8081   # should return renderer info
✅
If curl http://localhost:8081 returns a JSON response or renderer page, the image renderer is healthy.

5 Configure /etc/grafana/grafana.ini

bash — append/edit relevant sectionsbash
sudo tee -a /etc/grafana/grafana.ini <<'EOF'

[rendering]
server_url = http://localhost:8081/render
callback_url = http://localhost:3000/
renderer_token = my-renderer-secret-token
concurrent_render_request_limit = 10

[feature_toggles]
enable = accessControlOnCall externalServiceAccounts

[plugins]
allow_loading_unsigned_plugins = mahendrapaipuri-dashboardreporter-app
EOF

sudo systemctl restart grafana-server
ℹ️
The renderer_token value must exactly match authToken in /opt/grafana-renderer/config.json. Both are set to my-renderer-secret-token above — change both together in production.
Phase 3

Install & Use the Dashboard Reporter (PDF) Plugin

6 Install the Reporter Plugin

bashbash
PLUGIN_VER="1.8.0"   # check github.com/mahendrapaipuri/grafana-dashboard-reporter-app/releases
PLUGIN_URL="https://github.com/mahendrapaipuri/grafana-dashboard-reporter-app/releases/download/v${PLUGIN_VER}/mahendrapaipuri-dashboardreporter-app-${PLUGIN_VER}.zip"

wget -q $PLUGIN_URL -O /tmp/reporter-plugin.zip

# Unzip into Grafana's plugins directory
sudo unzip -q /tmp/reporter-plugin.zip \
  -d /var/lib/grafana/plugins/

# Normalize the folder name (strip version suffix if present)
sudo mv /var/lib/grafana/plugins/mahendrapaipuri-dashboardreporter-app-${PLUGIN_VER} \
        /var/lib/grafana/plugins/mahendrapaipuri-dashboardreporter-app \
        2>/dev/null || true

# Fix ownership — plugin must be readable by the grafana service user
sudo chown -R grafana:grafana /var/lib/grafana/plugins/mahendrapaipuri-dashboardreporter-app

sudo systemctl restart grafana-server
⚠️
You will see "plugin signature invalid" in journalctl -u grafana-server — this is expected and non-fatal, controlled by the allow_loading_unsigned_plugins line you added in Step 5.

7 Enable & Configure the Plugin in Grafana UI

  1. Log in as admin at http://localhost:3000
  2. Go to Apps → Dashboard Reporter App → Configuration
  3. Toggle the plugin to Enabled
  4. Set Grafana App URL → http://localhost:3000
  5. Under Layout, choose Grid (mirrors dashboard layout) or Simple
  6. Click Save

(Optional) Manual Service Account Token

If the externalServiceAccounts feature flag is unavailable:

  1. Administration → Service accounts → Add service account
  2. Role: Viewer
  3. Generate a token and paste it into the reporter plugin's Service Account Token field

8 Generate Your First PDF

Method A — Browser (simplest)

Browser Address Bar
http://localhost:3000/api/plugins/mahendrapaipuri-dashboardreporter-app/resources/report?dashUid=sales-demo-001

The browser downloads a PDF of the full dashboard while you are logged in as admin.

Method B — curl (automation & cron)

bashbash
GRAFANA_URL="http://localhost:3000"
DASH_UID="sales-demo-001"
TOKEN="<your-service-account-token>"
DATE=$(date +%Y%m%d)

curl -s -H "Authorization: Bearer $TOKEN" \
  "${GRAFANA_URL}/api/plugins/mahendrapaipuri-dashboardreporter-app/resources/report?dashUid=${DASH_UID}&from=now-7d&to=now&includePanelDataID=3" \
  -o "SalesReport-${DATE}.pdf"

echo "PDF saved: SalesReport-${DATE}.pdf"

Method C — Scheduled via cron

bash — add to crontab -ebash
# Every Monday at 07:00, generate and save the weekly report
0 7 * * 1 TOKEN="<token>" DASH="sales-demo-001" \
  curl -s -H "Authorization: Bearer $TOKEN" \
  "http://localhost:3000/api/plugins/mahendrapaipuri-dashboardreporter-app/resources/report?dashUid=$DASH&from=now-7d&to=now" \
  -o "/home/ubuntu/reports/sales-$(date +\%Y\%m\%d).pdf"
✅
includePanelDataID=3 appends the raw tabular data from the Product Sales Summary table panel to the end of the PDF, alongside its visual screenshot.
